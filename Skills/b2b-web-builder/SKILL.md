# 销售客户拜访展示工具 v2.0

销售人员在客户拜访时的信息展示工具。基于客户信息文档快速生成专业的多页展示网页。

## 核心用途

作为销售人员，在第一次见客户时，需要展示客户自己的各种信息。这个工具帮助你：
- 将客户信息文档转化为专业展示页面
- 在拜访时直接展示给客户看
- 展示客户自己的优势、案例、资质等信息
- 快速生成可在任何设备上打开的HTML网页

## 工作流程

### 1. 收集客户信息
- 客户名称
- 客户信息文档（.docx格式）
- 参考设计截图（可选）
- 主色调（可选）

### 2. 解析客户文档
从.docx文件中提取客户的：
- 基本信息（成立时间、团队规模、服务覆盖等）
- 产品/服务信息
- 客户案例和成功故事
- 资质认证和专利
- 竞争优势分析
- 销售渠道信息
- 组织架构
- 运营现状

### 3. 参照设计优化
如果提供了参考截图：
- 分析截图的UI风格
- 参照卡片设计、排版、配色
- 应用相同的交互效果

### 4. 生成展示网页
创建包含以下页面的单页应用：
- **首页**：客户名称、核心优势、关键数据
- **产品页**：客户的产品/服务系列、技术优势
- **案例页**：客户的成功案例、项目经验
- **资质页**：客户的认证、专利、荣誉
- **竞争页**：客户的竞争优势、市场地位
- **渠道页**：客户的销售渠道、服务网络

### 5. 输出展示网页
生成完整的index.html文件，可：
- 在浏览器中直接打开
- 部署到Netlify/Vercel
- 在客户拜访时展示
- 发送给客户查看

## 使用方式

### 基础用法
```
我要为客户准备拜访展示。
客户名称：派勒地垫
我有客户的详细信息文档（附件）
```

### 指定主色调
```
使用蓝色主题展示
```

### 参照设计
```
我有参考设计截图，请参照这些截图的样式
```

## 输出格式

生成的HTML文件包含：
- 完整的HTML5结构
- 内联CSS样式（无需外部文件）
- 原生JavaScript（无框架依赖）
- 响应式设计
- 所有资源都包含在单个文件中

## 技术特点

- **无依赖**：纯HTML/CSS/JavaScript，无需构建工具
- **响应式**：支持桌面、平板、手机
- **高性能**：轻量级，快速加载
- **易部署**：单个HTML文件，直接上传即可
- **易维护**：所有内容在一个文件中，易于修改
- **功能完整**：导航、页面切换、动画、交互都包含

## 配色方案

支持多种主题色：
- 蓝色系（默认）：#58a6ff - 专业科技感
- 绿色系：#2ecc71 - 环保生态感
- 紫色系：#9b59b6 - 创意创新感
- 橙色系：#e67e22 - 活力热情感
- 红色系：#e74c3c - 商业竞争感
- 青色系：#1abc9c - 清爽现代感

## 页面结构

每个页面都包含：
- 独立的背景渐变
- 主题色的标题和按钮
- 卡片式内容展示
- 详情内容区域
- 返回首页按钮

## 内容组织

### 首页
- Hero区域：客户名称、slogan、CTA按钮
- 核心优势：客户的关键优势卡片
- 客户信息：成立时间、团队规模、服务覆盖
- 联系方式：电话、邮箱、地址

### 产品页
- 产品系列：客户的产品/服务卡片
- 技术优势：客户的专利、认证、资质
- 应用案例：客户的典型项目详情

### 案例页
- 核心客户：客户的客户案例
- 项目详情：客户的成功故事、成果数据

### 资质页
- 认证资质：客户的ISO、高新企业等
- 专利信息：客户的专利详情
- 荣誉认可：客户的行业奖项

### 竞争页
- 竞争对手对比：客户vs竞争对手
- 客户优势分析：客户的竞争优势
- 市场地位：客户的行业地位

### 渠道页
- 销售渠道：客户的销售渠道
- 服务网络：客户的服务覆盖
- 发展方向：客户的渠道发展

## 最佳实践

1. **信息准备**：提供完整的客户信息文档
2. **参考设计**：如有参考截图，一起提供以获得更好的UI效果
3. **内容审核**：生成后检查所有内容是否准确
4. **部署测试**：部署后在不同设备上测试
5. **客户展示**：在拜访时直接展示给客户看

## 示例

输入：客户信息.docx + 参考截图 + 客户名称
输出：index.html（可在浏览器打开或部署展示）
